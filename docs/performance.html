Performance

http233-router-go targets high throughput with minimal hot-path allocations while providing a full Gin-like feature set.

Design

  • Radix tree — compressed prefix matching with static-before-param priority.
  • Context pooling — sync.Pool reuses Context, params, and handler buffers.
  • Pre-built chains — group middleware compiled at registration, not per request.
  • Fast path — no middleware → direct handler invocation.

Benchmarks

Benchmarks use direct TCP sockets (full HTTP read/write cycle), comparable to real server load:

go test -bench=. -benchmem ./...
go test -run TestFasterThanHttpRouter -v ./...

Comparison with httprouter

Automated test TestFasterThanHttpRouter compares parallel QPS against julienschmidt/httprouter with equivalent handlers.

ScenarioNotes
Static routes (100 routes)Comparable or faster QPS on parallel TCP benchmark
Parameter routes:id extraction with Context reuse
Business API (100+ routes)Middleware + groups included in integration suite

Trade-offs

http233 adds a Context layer and middleware chain compared to bare httprouter. The extra ~3 allocs/op buy Gin-style ergonomics (middleware, JSON helpers, groups). For routing-only micro-benchmarks with identical handlers, QPS remains competitive.
